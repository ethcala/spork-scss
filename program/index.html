Columns

Columns are made by using col-[number] where number is 1-12. Each column takes up 1/12 of the screen, so col-7 would take up 7/12 of the screen. You can have several columns next to each other with values that add up to 12 (such as one being col-5 and the other being col-7). It is not advised to use col to size a single element. The containing element should use the 'row' class. If you want all columns to be the same height, add the 'parent-height' class.

All of the below entries use col-7 paired with col-5.

Headers & Text

The header classes are used like default headers. They use the Crimson Text font (backup: Times, serif) and decrease by 2pt with each header size. The paragraph tags are also used like default p tags. They use the Zen Kaku Gothic Antique font (backup: Arial, sans serif). There is another more fun font option, Irish Grover. The font used in code tags is the Inconsolata font (backup: Lucida Console, monospace).

You can change text properties in several ways. There are classes to change the weight and size of fonts as well as styling and the font itself. Please note that the light weight does not work well with most of these fonts. Weight options are classes starting with 'txt-w-' and have the options of light, normal, semibold and bold. Size options start with 'txt-s-' and have the options of lg, md, and sm. Decoration options start with 'txt-d-' and have the options of line-thru and line-under. Font can be changed with 'txt-font-' followed by header, basic, code or fun. The color can be changed with 'txt-black' or 'txt-white'. These options can all be applied if you use multiple different effects. Text can be aligned with txt-a-[center, left, right].

Links look as shown in the example with an underline by default, but you can remove the styling for buttons, navbars, etc by using the 'no-deco' class.

<p class='txt-w-semibold txt-d-line-under'>Test text</h6>

Lists

The lists have new default styling shown to the left. They are used as usual and can be modified by any text or background modifiers. They can also be styled with 'list-unstyled' and 'list-bullet', shown in the example.

Images

Images use the default tag and include a secondary colour border. There are transition classes designed for images that cause them to expand slightly when moused over. These can be edited in the scss and new ones with new default sizes can be created, simply import 'transitions', create a class and include 'tr-larger(width in px, height in px)' and add it to your image. The default transition classes are 200x200 ('tr-sq-larger'), 300x200 ('tr-rect-larger'), and 100x100 ('tr-smsq-larger').

<image src='https://via.placeholder.com/100' class='tr-smsq-larger'></image>

Borders

Bordered objects need to have at least one of five classes that indicate the location of the borders. These are border-right, border-left, border-top, border-bottom, and border-all. Double thickness versions of these are followed by -thick in the class name.

Borders can have effects such as dotted or dashed and come in the black or white colors. They can also come in the darker secondary color used for the text by default. Border classes start with 'border-'. Possible endings are dashed, dotted, round, black, white, or dark (text color).

<div class='border-top border-bottom border-black'>Test border</div>

Spacing

You can set the margins by using m[position]-[multiplier]. All margins start with a base of .5rem, and are multiplied by the multiplier. m-0 would mean there are no margins. The positions are left (l), right (r), top (t), bottom (b), left and right (x), and top and bottom (y). ml-3 would give a left margin of 1.5rem. The padding system works exactly the same. 'w-' 100, 90, and 50 exist to give more control over element width.

Alignment

Alignment is done by using align-left, align-right, align-top or align-bottom. You can also use center-x to center horizontally and center-y to center vertically. To make two divs the same height as a containing div for alignment and centering purposes, you can give each of the contained divs the parent-height class. The row class will make contained elements line up horizontally while the stack class will make them line up vertically using flexbox. Rows automatically set at 100% width and center on the y axis.

Stack or row can be followed by -xs, -sm, -md, or -lg to cause them to only take effect when the screen width is lower than the specified size preset. If an element has the stack-sm class, it will stack only when the screen width is lower than 800px. You can use hide-xs, -sm, -md, or -lg to hide an element when the screen is a certain size, such as the example: There are 5 boxes until the xs limit is hit, then there are 3.

Tables

Tables are created using the standard table, tr, th and td classes. Each has been customized. The padding for head elements is slightly larger than in data elements and the text in those elements are bold. The padding on the right is the largest. Tables have a small default margin on all sides.

The examples for this section show the way tables can be altered with classes. All classes are placed in the outer table tag. The available classes are 't-dark', 'alternate-colors' and 'bold-top', all of which can be used together. All possible combinations of these tags are shown in the examples. The regular table colouring uses the main colour, while the dark table uses the secondary colour. Since usage of tables is not a new concept, only the classes are shown in the code sample. The sample would result in the bottom right table shown.

<table class='t-dark bold-top alternate-colors'>

...

Other Unique Tags

The code element is replaced by a div element with the 'code-light' or 'code-dark' class. You can nest p tags inside. This usage won't be demonstrated because that would cause a dimensional break. We think you can figure it out on your own though. Here's what they both look like, for reference:

You may use a JSON file to create a tip/quote that rotates through several randomly. You must have the quotes you wish to display in a file called data.json and use a div with the msg-holder class (nothing else inside it). It automatically formats.
