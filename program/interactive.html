Headers & Footers

Navbars have the class options of 'nav-light' and 'nav-dark', shown in the example. A nested div should be tagged 'item-holder' and contain all p tags. Nested items must use 'nav-title' for the first one, the others should have 'nav-item' with the active one tagged 'active'. Disabled items should be tagged 'disabled'. Items by default are pushed to the left, but you may tag the top element with 'items-right' to push them right. You will also need to contain all items to be pushed to the right with a div with a 'right-holder' class. Adding the 'center-x' class will center all text.

Navbars can have the 'nav-fixed' class to fix to the top. There is a 'blank-top' div class to place above content to fit the fixed navbar. You may need the 'front' class on the fixed navbar to keep it in front of everything else.

To create responsive navbars, you will need an a tag under the item-holder div. This should have the properties href='javascript:void(0);' and class='nav-icon', and contain a nested icon with the class 'fa fa-bars'.

Headers and footers are used as normal with new default styling. Add a 'header-light' or 'footer-light' class for a light version. Headings are the same size as paragraphs by default and both are centered. Please don't put a footer in any divs or containers because it will cry. Headers are fine though.

<div class='navbar nav-dark mt-6 items-right'>

  <div class='item-holder'>

    <p class='nav-title'>Nav Dark</p>

    <div class='right-holder'>

      <p class='nav-item active'>Active</p>

      <p class='nav-item'>Inactive</p>

      <p class='nav-item disabled'>Disabled</p>

    </div>

  </div>

  <a href='javascript:void(0);' class='nav-icon'>

    <i class='fa fa-bars'></i>

  </a>

</div>

Buttons

Buttons are called with a div with the class 'button-' followed by dark, light, main, red, green or yellow. Text properties can also be altered in the div, such as in the example code.

Buttons start at 120px wide and 35px tall. The paragraph font and size are used. Links will not be styled by default. Buttons built with this class can use p tags, but will work without them.

<div class='button-main txt-white fw-semibold'>Test button</div>

Forms

All form elements are remade. Labels can have text property classes added to them. Forms can contain divs with the row or stack classes as needed. The default button tag is styled in the same way as the submit button.

Forms will by default take up 50% of their containing element. The basic font is used by default for all elements in a form.

<form>

  <div class='row stack-sm'>

    <label class='txt-w-semibold'>Text:</label>

    <input type='text'>

  </div>

  <div class='stack'>

    <label class='txt-w-semibold'>Radio:</label>

    <div class='row'>

      <input type='radio' name='ex' id='opt-1a'>

      <label for='opt-1a'>Option 1</label>

    </div>

    <div class='row'>

      <input type='radio' name='ex' id='opt-2a'>

      <label for='opt-2a'>Option 2</label>

    </div>

    <div class='row'>

      <input type='radio' name='ex' id='opt-3a'>

      <label for='opt-3a'>Option 3</label>

    </div>

</form>
